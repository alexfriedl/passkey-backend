keepWebsiteOpen Test

Teste alle Finish-Callback Szenarien

Voraussetzung

  • PasskeyGuard App muss installiert sein
  • Starte zuerst einen Flow mit dem "Start Flow" Button
  • Dann teste die Finish-Callbacks

0. Flow starten

Öffnet WebView - muss vor den Finish-Tests ausgeführt werden

Start Flow (mit returnURL) Start Flow (ohne returnURL)

Fall 1: keepWebsiteOpen=true

Number Matching - User muss finale Seite sehen

Erwartetes Ergebnis:
WebView bleibt offen + Banner erscheint oben
Schema Link Universal Link

passkeyguard://finish?keepWebsiteOpen=true

Fall 2: keepWebsiteOpen=false + returnURL

WebRTC / Registration - Auto-Close und Redirect

Erwartetes Ergebnis:
WebView schließt + returnURL (success.html) wird geöffnet
(Nur wenn Flow mit returnURL gestartet wurde!)
Schema Link Universal Link

passkeyguard://finish?keepWebsiteOpen=false

Fall 3: keepWebsiteOpen=false (ohne returnURL)

WebView schließt ohne Redirect

Erwartetes Ergebnis:
WebView schließt nur (kein Redirect)
(Starte Flow OHNE returnURL um diesen Fall zu testen)
Schema Link

Fall 4: Kein keepWebsiteOpen Parameter

Backwards Compatibility - verhält sich wie keepWebsiteOpen=false

Erwartetes Ergebnis:
Wie Fall 2/3 - WebView schließt (+ returnURL falls vorhanden)
Schema Link (ohne Param) Universal Link (ohne Param)

passkeyguard://finish
