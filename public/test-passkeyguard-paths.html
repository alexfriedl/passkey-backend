Test: Universal Link Pfade

Neue Pfade:
/passkeyguard/open - Startet Safari-Flow
/passkeyguard/finish - Beendet Flow (Banner/Schliessen)

Domain Einstellung

iOS blockiert same-domain Universal Links in SFSafariViewController. Cross-domain sollte funktionieren.





1. Test: /passkeyguard/open

Dieser Universal Link startet den PasskeyGuard Safari-Flow.

Universal Link:
/passkeyguard/open testen

2. Test: /passkeyguard/finish

Dieser Universal Link beendet den Flow.

Universal Link:
/passkeyguard/finish testen

3. Kompletter Flow-Test

Oeffnet eine Registrierungsseite die am Ende /passkeyguard/finish aufruft.

Hinweis: Diese Seite muss auf www.appsprint.de deployed sein, damit Universal Links funktionieren.
Kompletten Flow starten
Voraussetzungen:
  • App muss www.appsprint.de in allowedDomains haben
  • AASA muss /passkeyguard/* Pfade enthalten
  • Test auf echtem iOS-Geraet (Simulator unterstuetzt keine Universal Links)
