Non-Discoverable Login

Was ist Non-Discoverable?

Username erforderlich!

Der Server schickt eine spezifische Credential-ID. iOS sucht nur nach diesem einen Passkey.

allowCredentials: [{ id: "abc123...", type: "public-key" }]

Technischer Ablauf:
  1. Benutzer gibt Username ein
  2. Client ruft /api/login mit Username auf
  3. Server sucht User und antwortet mit dessen allowCredentials
  4. iOS sucht NUR nach dieser spezifischen Credential-ID
  5. Client sendet Assertion zur Verifikation
